# DocPilot E2E 业务场景测试矩阵 v1.0

> **目的**：从**真实用户业务流**出发，梳理所有应测场景（正常 + 异常 + 边界 + 性能 + 安全）
>
> **基线**：设计文档 `design/03-conversation-flow.md` + `design/04-ui-design.md` + `design/02-data-model.md`
>
> **方法**：先确认业务流 → 从流拆分正常/异常 → 每个 case 标注优先级 + 类型
>
> **生成日期**：2026-07-20
>
> **核心原则**（老大 11:52 指示）：**先看设计→确认业务流→如无补业务流→从业务流拆分所有场景→不要限定写多少**

### BF-01 模式 A（用户主动输入）

**业务流步骤**（来自 04-ui-design § 4.1）：
1. 用户进入页面（无历史 → 模式 C）

### BF-06 保存周报

**业务流步骤**（来自 04-ui-design § 4.6）：
1. PreviewPane 点击 [保存周报]
2. reportStore.isSaving = true
3. POST /api/v1/reports { sessionId, templateId, title, content, summary, metadata }
4. 后端 upsert（按 session_id 唯一约束）
5. 返回 ReportResponse（含 id）
6. reportStore.currentId = id
7. reportStore.isDirty = false
8. reportStore.status = 'saved'
9. ElMessage.success('保存成功')
